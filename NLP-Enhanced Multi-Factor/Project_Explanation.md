# NLP-Enhanced Multi-Factor Strategy Project Overview

## 1. 项目核心逻辑详解

这个项目的核心思想是：**市场价格不仅由财务数据驱动，还受到投资者情绪（舆情）的强烈影响**。

### 逻辑链路
1.  **信息输入 (Input)**:
    *   **市场数据**: 股票的历史价格 (OHLCV) -> 反映市场当前的交易状态。
    *   **非结构化数据**: 财经新闻、公告、研报 -> 包含潜在的利好/利空信息，往往领先于价格变动。

2.  **信息处理 (Processing)**:
    *   **NLP 模型 (大脑)**: 使用预训练的深度学习模型 (如 BERT/Roberta) 阅读新闻。它不只是看关键词，而是理解上下文语义。
    *   **情感量化**: 模型将文本转化为一个 **情感分数 (Sentiment Score)**。例如：
        *   "公司业绩大增，超预期" -> Score: 0.9 (极度积极)
        *   "工厂发生火灾，停产整顿" -> Score: -0.8 (消极)
    
3.  **策略生成 (Strategy)**:
    *   **Alpha 因子**: 将情感分数作为一个新的 Alpha 因子。
    *   **交易规则**:
        *   当情感分数 > 阈值 (如 0.3) -> 认为市场情绪向好 -> **买入/持有**
        *   当情感分数 < 阈值 (如 -0.3) -> 认为市场情绪恶化 -> **卖出/空仓**

4.  **回测验证 (Backtest)**:
    *   在历史数据上模拟上述过程，验证如果我过去几年这么做，能不能赚钱？赚多少？风险多大？

### 为什么这样做有效？
*   **信息优势**: 机器阅读速度远超人类，能瞬间处理海量新闻。
*   **情绪溢价**: A股散户多，情绪波动大，舆情因子往往有较高的超额收益 (Alpha)。

## 2. 免费新闻数据源调研

获取高质量、长周期的历史新闻数据是量化项目的最大难点。以下是几种免费或低成本的方案：

### A. AKShare (当前使用)
*   **接口**: `stock_news_em` (东方财富), `stock_info_global_cls` (财联社)
*   **优点**: 免费，Python 直接调用，对接方便。
*   **缺点**: **历史数据极少**。通常只能获取最近几页（几天到几周）的新闻，无法满足长周期回测需求。

### B. Tushare (积分制)
*   **接口**: `news` (新闻联播), `cctv_news`
*   **优点**: 数据规范，有较老的历史数据。
*   **缺点**: 需要积分（注册送少量，高校学生认证送积分），部分高级新闻接口需要付费。
*   **建议**: **推荐高校学生尝试认证 Tushare Pro**，获取积分后不仅能下新闻，还能下非常规范的行情数据。

### C. GNews (Google News API 非官方)
*   **工具**: `pip install gnews`
*   **优点**: 全球新闻，支持按关键词和时间段搜索。
*   **缺点**: 对中文 A 股个股的支持不如国内财经网站精准，且容易被反爬限制。

### D. 自建爬虫 (Advanced)
*   **目标**: 搜狐财经、新浪财经、巨潮资讯网 (公告)。
*   **方法**: 使用 `Scrapy` 或 `Selenium` 定向爬取。
*   **优点**: 数据完全掌控，可回溯很久。
*   **缺点**: 技术门槛高，维护成本高，有法律风险和反爬风险。

### E. 替代方案：基于公告和研报摘要
*   **思路**: 不用全网新闻，只用公司发布的**公告摘要**或**研报标题**。
*   **AKShare 接口**: `stock_report_em` (研报), `stock_notice_report` (公告)。
*   **优点**: 相比新闻，公告和研报的历史数据更容易获取且更有结构化。
